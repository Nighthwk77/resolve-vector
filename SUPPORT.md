# Support

## Before opening an issue

1. Update to the newest public release.
2. Read [Getting Started](docs/GETTING_STARTED.md) and [Troubleshooting](docs/TROUBLESHOOTING.md).
3. Check existing issues for the same behavior.
4. Reproduce the problem with the smallest safe project or example possible.

## Bug reports

Use the repository's **Bug report** form. Include the app version, macOS version, expected result, actual result, and reproducible steps.

Never attach API keys, `.vibe/secrets.json`, private source code, or unreviewed logs.

## Feature requests

Use the **Feature request** form and explain the problem the feature would solve. Requests are evaluated against Resolve Vector's local-first, inspectable, approval-oriented design.

## Provider account support

Billing, rate limits, subscriptions, model availability, and account access are controlled by the provider. Contact that provider for account-specific support.
