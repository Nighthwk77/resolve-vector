# Security Policy

## Supported versions

Security fixes are provided for the newest public release. Update to the newest release before reporting behavior that may already be corrected.

## Report a vulnerability privately

Do not open a public issue for vulnerabilities, exposed credentials, authentication problems, unsafe command execution, or reports containing private source code.

Use GitHub's **Report a vulnerability** option in the Security tab of this repository. Include:

- the Resolve Vector version and macOS version;
- a concise description of the impact;
- reproduction steps using non-sensitive sample data;
- relevant logs with credentials and private code removed;
- whether you believe exploitation is active.

You will receive acknowledgment through the private GitHub advisory. Please allow time to investigate before publishing details.

## Credential exposure

If a provider key may have been exposed, revoke or rotate it immediately through that provider. Do not wait for a software investigation before protecting the account.
